    • ESP32-S3: Dual-core, optimized for AI/ML, Wi-Fi + Bluetooth.

Memory Chips

These are programmable storage memory chips. They store large amounts of data but cannot run code on their own. They require an external processor to access and process the data. NO executables.

EEPROM (24XX Series, I²C (Inter-Integrated Circuit) Interface/Protocol):
  • 24C02, 24C04, 24C08, 24C16, 24C32, 24C64, 24C128, 24C256, 24C512
    • AT24C256 – 256Kb EEPROM memory for data storage.

 

SPI Flash Memory (25XX Series, SPI(Serial Peripheral Interface) Interface/Protocol):
  • 25Q32, 25Q64, 25Q128, 25Q256
    • Winbond W25Q64 – 64Mbit (8MB) SPI Flash, used in ESP modules.

 

NOR & NAND Flash (Larger Storage, Parallel interface):
  • Spansion GL512S110HA02
  • Winbond W25N01 (1Gb NAND Flash)
  • SK Hynix H26M42003GMR (e-NAND)

 

e-NAND 
Raw storage, no controller in the package. It requires an external controller to manage data handling and mimic the behavior of an eMMC.
  • SK Hynix H26M42003GMR

 

eMMC

System-on-Chip (SoC)

A complete computing system on a single package with a CPU, GPU, and peripherals. It does NOT have large internal storage like memory chips and relies on external storage (eMMC, UFS, SSD) to run an operating system and applications.

Packaging Style

 

Packaging type can hold by microcontrollers, memory Chips, Logic ICs, Op-Amps, Microprocessors. 

  • DIP (Dual In-Line): Pins on two sides, breadboard-friendly: ATmega328P, 24C256 EEPROM
  • QFP (Quad Flat Pack): Pins on all four sides of the chip    ATmega2560, ESP32-S3
  • BGA (Ball Grid Array): Solder balls underneath, very compact. SK Hynix H26M42003GMR (e-NAND)

There are only two main mounting types for electronic components:

1. Through-Hole Technology (THT) or Insertion Mounting or Pin-In-Hole

  • Pins go through holes in the PCB and are soldered underneath.

2. Surface Mount Technology (SMT)

  • Components sit on the surface of the PCB and are soldered directly.

ðŸš« No other standard mounting types exist for general PCBs.

  • Some niche techniques (like COB and Press-Fit) are just variations of SMT or special cases.

 

 

 

Communications Protocols 

 

SPI - Serial Peripheral Interface: Requires four wires: MISO, MOSI, SCLK, and CS. Communication is faster, but covers short distances, Supports single-master and multi-slave communication.

IIC - Inter-Integrated Circuit: Requires 2 wires: SCL and SDA. Communication is slower, but covers long distances, Supports multi-master and multi-slave communication.

UART - Universal Asynchronous Receiver/Transmitter. Requires 2 wires: TX and RX. Communication is slower, but covers long distances. Supports point-to-point communication.

CAN - To ensure the security of transmission a large number of means are used to ensure timely and accurate data transmission. At the same time, the identification code is used to solve the problem of bus conflicts. This ensures that the high-priority devices communicate first. That's why CAN can be used in industries and automobiles that require high levels of safety.
