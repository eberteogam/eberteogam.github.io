Work Streams / Common Path / Business Drivers

Work Streams

  • Mgmt Scoping: Proper documentation before deploying
  • Platform Network: Scale Virtual Appliances (VA - Reconfigured linux machines for SP's use) accordingly.
  • Data Identity: Place external identities in Authoritative Systems (Separated db for external identities)
  • Application: Aim to build centralized workflows ensuring scoped apps are using them.

Business Drivers

  • Information Security: + Security - audit funding. CP → TC → AM → LM → PM
  • Financial Operating: - operation cost + profit margins. CP → PM → LM → AM → TC
  • Operation Productivity: + productivity. CP → PaM → AR → AM → LM

Common Path CP

  • Identity Foundation: Authoritative Systems in scoped apps, key applications
  • Advanced Analytics: Verify data is correct and clear
  • Access Insights: Capture access history ASAP to hlup AI/ML

Abbreviations

  • TC - Targeted Cert.: IGA access review campaigns, process can be automated by AI
  • AM - Access Modeling: Bundles the access services of business or birth role access
  • LM - Lifecycle Mgmt: Leverage access models to automate lifecycle events
  • PM - Policy Modeling: Surface violations that need to be reviewed and actioned
  • PaM - Password Mgmt: Password reset calls, self service implementation
  • AR - Access Request: Enhanced with AI recommendations to expedite user requests without governance control

Identity Security

  • Least Privilege: People only have the access they absolutely need
  • Automated Provisioning: Seamless employee access changes in their business lifecycle
  • Access Certifications & Separation of duties
